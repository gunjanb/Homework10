Strategies for improving load time of web sites

To know how we can improve the load time of web sites first we need to understand what exactly the Page Speed is. This term essentially refers to the speed with which Web Pages or Media Content is downloaded from website hosting servers and displayed onto the requesting Web browser. So, the Page Load time is the duration between clicking the link and displaying the content of the web page on the requested browser.

Does the Speed of Website really matters?

Amazon found out that a 1sec delay in page load time yields:

  • 11% fewer page views
  • 16% decrease in customer satisfaction
  • 7%loss in conversions.

Amazon found out that even if the website speed is improved by 100 milliseconds it results in increases revenue of 1%. So, did Walmart, who found a 2% increase in conversions for every 1 second of improvement. Clearly, we can say that speeding up of your website is critical. In below section, few techniques are discussed to improve the same.

Techniques to improve the Site Speed:

1.Minimize HTTP Requests:

According to Yahoo, 80% of a Web page’s load time is spent downloading the different parts of the page like images, stylesheets, scripts, Flash, etc. An HTTP request is made for each one of these elements, so the more on-page components, the longer it takes for the page to render.If that is the case we can do it by simplifying our design by:

  • Streamline the number of elements on your page.
  • Use CSS instead of images whenever possible.
  • Combine multiple style sheets into one.
  • Reduce scripts and put them at the bottom of the page.

2.Enable Compression:

Large pages (usually 100kb and more) are bulky and slow to download. The best way to speed their load time is to zip them and technique for doing this called compression.

Compression reduces the Bandwidth of the page, thereby reducing HTTP response. We can perform zipping by a tool called Gzip. According to Yahoo compression can reduce the load time by 70%. Most of the web servers can compress files in Gzip format before sending the for download, either by calling a third-party module or using built in routines

3.Enable Browser caching:

When user visits the website for the first time they have to download the HTML document, stylesheets, JavaScript files and images before it can use the web page. That may take as many as 30 components and 2.4 seconds to download.

Once the page has been loaded and the different components stored in the user’s cache, only a few components needs to be downloaded for subsequent visits. Usually 40-60% of daily visitors come in with empty cache, so its critical that you make your page fast for these first -time visitors but we need to enable caching to save time off for subsequent visits.

4.Minify Resources:

Unnecessary piece of code adds to the size of your page, it’s important that you eliminate extra spaces, line breaks, and indentation in your code so your pages are as lean as possible It also helps in minifying our own code. Few ways to do it as:

  • To minify HTML, you can use PageSpeed Insights Chrome Extension to generate an optimized version of your HTML code. Run the analysis against your HTML page and browse to the ‘Minify HTML’ rule. Click on ‘See optimized content’ to get the optimized HTML code.
  • To minify CSS, you can try cssmin.js
  • To minify JavaScript, try the Closure Compiler or JSMin. You can create a build process that uses these tools to minify and rename the development files and save them to a production directory.

5.Optimize images:

To Optimize images, we can do it by taking care of image sizes, format and src attributes.

Image Size:

Oversized images take longer time to load, so it’s important that we keep web images as small as possible. Use image editing tools to:

  • Crop your images to the correct size. For instance, if your page is 570px wide, resize the image to that width. Don’t just upload a 2000px-wide image and set the width parameter (width=”570”). This slows your page load time and creates a bad user experience.
  • Reduce color depth to the lowest acceptable level.
  • Remove image comments.

Image Format:

  • JPEG is the best option.
  • PNG is also good, though older browsers may not fully support it.
  • GIFs should only be used for small or simple graphics (less than 10×10 pixels, or a color palette of 3 or fewer colors) and for animated images.
  • Do not use BMPs or TIFFs.

Src Attribute:

Once we got the size and format right, we should avoid empty image src codes.When there’s no source in the quotation marks, the browser makes a request to the directory of the page or to the actual page itself. This can add unnecessary traffic to your servers and even corrupt user data.

6.Optimize CSS Delivery:

CSS have the style requirements for the web page and website accesses this information in one of the two ways: -

  • Inline, which is inserted in the HTML document itself.
  • In an external file, which loads before your page renders. This is loaded in the head of your HTML Code and is preferable as it reduces the size of your code and creates fewer code duplications.

7.the Reduce number of plugins you use on your site:

Too many plugins slow your site, create security issues, and often cause crashes and other technical difficulties.

8.Reduce redirects:

Redirects create additional HTTP requests and increase load time. So we should keep them to a minimum.

9.Let the Network Carry the Load:

A content delivery network is a way of taking a websites static files, like CSS, images, and JavaScript, and delivering them through web servers that are closer to the user’s physical location. Shorter proximity amounts to faster load time.

10. Review your hosting service plan:

If you have gone through the checklist above and still find that web performance is impacted, check with your hosting provider to see whether you have shared or dedicated hosting. A shared hosting account often involves sharing server space with dozens of other companies where website speed is impacted by the number of people using the servers. If this is the case, then it may be time to consider a dedicated plan where you have sole access to the server.

11.Code your site for mobile-first quality and speed:

The strategy should be to code for mobile users first and then progressively enhance the experience for tablets and desktops. Doing so will help reduce the number of unnecessary dependencies.
